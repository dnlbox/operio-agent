# Developer Agent Instructions

Welcome to the **Operio Autonomous Mall Operations Agent** codebase. This file provides guidelines and instructions for AI developer agents modifying or working within this repository.

---

## 1. Directory Structure

Ensure any new files align with this structure:
*   `docs/` — Product, architectural, and hackathon documentation.
*   `docs/diagrams/` — D2 diagram source files (`.d2`) and rendered images (`.svg`).
*   `src/` — Source code for the backend and frontend.
    *   `src/backend/` — Express/Node.js API server, agent routing, and MCP clients.
    *   `src/frontend/` — Single-page dashboard (HTML/CSS/JS).
*   `tests/` — Mock dataset generators, test suites, and evaluation runs.

---

## 2. Guidelines for Agents

### A. Code Style & Tech Stack
*   **Backend:** Node.js with **TypeScript** and **Hono** framework, built and managed using **pnpm**.
*   **Path Aliases:** Use path mappings (`@/*` pointing to `src/*`) for all local imports to avoid deep relative paths (e.g., `import { db } from '@/backend/db'`).
*   **TSDoc/JSDoc Guidelines:** Follow Google's styling guidelines:
    *   Do not repeat types in JSDoc inside TypeScript files (let TypeScript enforce types; use comments to describe semantic behavior).
    *   Document all public classes, methods, and interfaces.
*   **Frontend:** Vanilla HTML, Vanilla CSS, and modern clientside Javascript. Do not pull in large frontend frameworks unless requested by the user.
*   **Styling:** Focus on clean, modern, and beautiful design layouts following "The Precision Authority" tokens (glassmorphic cues, subtle gradients, and transition animations).

### B. Diagram Conventions
*   All diagrams must use **D2** syntax.
*   Keep diagram sources in `docs/diagrams/*.d2`.
*   Compile `.d2` files to `.svg` using:
    ```bash
    d2 docs/diagrams/<filename>.d2 docs/diagrams/<filename>.svg
    ```

### C. MCP Integration
*   Ensure agent actions are written as structured tool schemas.
*   The orchestrator must route search calls to the **Elastic MCP client** and operational/transactional changes to the **MongoDB MCP client**.
*   All agent logic execution must be instrumented for tracing in **Arize Phoenix**.

---

## 3. Common Development Commands

*   `npm run dev` — Starts the backend server and serves the frontend dashboard.
*   `npm run test` — Runs the unit test suite.
*   `npm run evaluate` — Triggers the evaluation harness against the mock scenario dataset.
*   `npm run seed` — Resets and seeds the mock databases (MongoDB and Elastic indexes).
